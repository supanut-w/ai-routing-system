# PERSISTENT MEMORY

## Objective
This directory stores long-term knowledge, "lessons learned," and architectural decisions that should persist across sessions.

## Memory Structure
- `lessons_learned.md`: Post-mortem insights from completed tasks.
- `architectural_decisions.md`: Record of major system changes and why they were made.
- `user_preferences_history.md`: Tracking evolution of user DNA and style.

## Interaction Protocol
1. **Record**: After completing a major phase, the Reviewer or Router should summarize key learnings.
2. **Recall**: Before starting a new task, the Researcher should check memory for relevant past context.
