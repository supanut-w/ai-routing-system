# TOOL: HTML REPORTER (v1.0)

## Objective
Enable the generation of high-fidelity, interactive HTML reports with professional styling, animations, filtering, sorting, and rendered flows. This tool provides a zero-install CDN stack for portability across all AI providers and laptops.

## The RIR Tech Stack (CDN-Based)
1. **Styling**: [Tailwind CSS](https://tailwindcss.com/)
2. **Interactivity**: [Alpine.js](https://alpinejs.dev/)
3. **Flows/Diagrams**: [Mermaid.js](https://mermaid.js.org/)

## Boilerplate Template
The Implementer MUST use this boilerplate as the foundation for all `report.html` files:

```html
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Project Report</title>
    <!-- Tailwind CSS CDN -->
    <script src="https://cdn.tailwindcss.com"></script>
    <!-- Alpine.js CDN -->
    <script defer src="https://cdn.jsdelivr.net/npm/alpinejs@3.x.x/dist/cdn.min.js"></script>
    <!-- Mermaid.js CDN -->
    <script src="https://cdn.jsdelivr.net/npm/mermaid/dist/mermaid.min.js"></script>
    <script>mermaid.initialize({ startOnLoad: true });</script>
</head>
<body class="bg-gray-50 text-gray-900">
    <div class="max-w-6xl mx-auto p-8" x-data="{ tab: 'overview' }">
        <!-- Header -->
        <header class="mb-12">
            <h1 class="text-4xl font-bold text-blue-600">Interactive Project Report</h1>
            <p class="text-gray-500">Generated by AI Routing System v4.8.0</p>
        </header>

        <!-- Navigation Tabs -->
        <nav class="flex space-x-4 mb-8 border-b">
            <button @click="tab = 'overview'" :class="tab === 'overview' ? 'border-blue-600 text-blue-600' : 'border-transparent text-gray-500'" class="pb-2 border-b-2 font-medium">Overview</button>
            <button @click="tab = 'data'" :class="tab === 'data' ? 'border-blue-600 text-blue-600' : 'border-transparent text-gray-500'" class="pb-2 border-b-2 font-medium">Data & Analysis</button>
            <button @click="tab = 'flows'" :class="tab === 'flows' ? 'border-blue-600 text-blue-600' : 'border-transparent text-gray-500'" class="pb-2 border-b-2 font-medium">Workflows</button>
        </nav>

        <!-- Content -->
        <main>
            <!-- Tab: Overview -->
            <div x-show="tab === 'overview'" x-transition>
                <div class="bg-white p-6 rounded-lg shadow-sm border">
                    <h2 class="text-2xl font-semibold mb-4">Project Summary</h2>
                    <p>Core findings and high-level mission status.</p>
                </div>
            </div>

            <!-- Tab: Data -->
            <div x-show="tab === 'data'" x-transition>
                <div class="bg-white p-6 rounded-lg shadow-sm border overflow-x-auto">
                    <h2 class="text-2xl font-semibold mb-4">Data Analysis</h2>
                    <!-- Example of Alpine-based filtering/sorting logic would go here -->
                    <table class="min-w-full divide-y divide-gray-200">
                        <thead class="bg-gray-50">
                            <tr>
                                <th class="px-6 py-3 text-left text-xs font-medium text-gray-500 uppercase tracking-wider">Metric</th>
                                <th class="px-6 py-3 text-left text-xs font-medium text-gray-500 uppercase tracking-wider">Value</th>
                            </tr>
                        </thead>
                        <tbody class="bg-white divide-y divide-gray-200">
                            <tr>
                                <td class="px-6 py-4 whitespace-nowrap font-medium text-gray-900 text-sm">Example Metric</td>
                                <td class="px-6 py-4 whitespace-nowrap text-sm text-gray-500">1,234.56</td>
                            </tr>
                        </tbody>
                    </table>
                </div>
            </div>

            <!-- Tab: Flows -->
            <div x-show="tab === 'flows'" x-transition>
                <div class="bg-white p-6 rounded-lg shadow-sm border">
                    <h2 class="text-2xl font-semibold mb-4">System Flows</h2>
                    <div class="mermaid">
                        graph TD
                        A[User] -->|Prompt| B(Router)
                        B -->|Plan| C(Strategist)
                        C -->|Logic| B
                        B -->|Action| D[Team]
                    </div>
                </div>
            </div>
        </main>
    </div>
</body>
</html>
```

## Tool Protocol (Standard)
1. **Check**: Verify if the mission requires a rich report (RIR).
2. **Strategy**: The Strategist MUST specify the report schema in the Routing Plan.
3. **Execution**: The Implementer constructs the `report.html` using the template above, embedding data gathered by the Researcher.
4. **Validation**: The Reviewer vets the HTML for technical correctness, and the Auditor verifies alignment with User Intent.
